## 测试目标

验证 wechat-writer agent 在 AI 主题下是否能够：

1. **正确识别主题**：识别用户请求是 AI 主题，应用 AI 主题风格
2. **单标题结构**：创建符合 AI 主题风格的单标题
3. **比喻驱动**：使用贯穿全文的比喻
4. **阶段划分**：使用明确的阶段编号
5. **互动设计**：设计互动话题和投票选项
6. **风格一致性**：与现有 AI 文章风格保持一致

## AI 主题风格特征

### 标题风格

- **格式**：单标题（引导词 + 核心主题）+ emoji
- **引导词**："一文看懂"、"深度解析"、"全面了解"
- **Emoji**：🚗, 🤖, 💡, 🚀, 🧊, 🎯

### 文章结构

- **开头**：Blockquote（💡）+ 【导语】+ 比喻
- **章节**：阶段编号（第一阶段、第二阶段...）+ emoji + 阶段名称
- **结尾**：【结语】+ 范式转移 + 展望未来
- **互动**：【互动话题】+ 投票选项 + 评论区邀请

### 内容特点

- **比喻驱动**：使用贯穿全文的比喻
- **列表说明**：使用列表说明优势和痛点
- **配图建议**：每个阶段都有配图建议

### 视觉元素

- 大量 emoji 作为标记和装饰
- 列表标记（✅, ❌, 🗺️, ⚠️, 🧰）
- 配图建议（描述性，非固定链接）

### 互动元素

- 【互动话题】标记
- 3-5 个投票选项
- 评论区邀请

## 验证流程

### 步骤 1：准备测试

1. 选择测试用例
2. 准备输入信息
3. 准备参考文档（AI 主题风格规范、AI 文章模板）

### 步骤 2：执行测试

1. 调用 wechat-writer agent
2. 明确指定主题类型为 "AI"
3. 提供测试任务和输入信息
4. 等待 agent 生成文章

### 步骤 3：验证输出

1. **自动验证**（如果有脚本）：
   - 检查标题格式
   - 检查阶段编号
   - 检查字数
   - 检查 emoji 使用

2. **人工验证**：
   - 对照 AI 主题风格规范逐项检查
   - 与现有 AI 文章对比风格
   - 检查内容质量

### 步骤 4：评分与反馈

1. 根据验证标准逐项打分
2. 计算总分
3. 记录问题和建议
4. 生成测试报告

---

## 评分标准

### 综合评分计算

**综合评分 = 标题 × 20% + 结构 × 30% + 内容 × 25% + 视觉 × 15% + 互动 × 10%**

### 评分等级

| 等级 | 分数范围 | 说明 |
|------|----------|------|
| **优秀** | 90-100 | 完全符合要求，可直接发布 |
| **良好** | 80-89 | 基本符合要求，少量修改后可发布 |
| **合格** | 70-79 | 部分符合要求，需要较多修改 |
| **不合格** | < 70 | 不符合要求，需要重新创作 |

### 测试通过标准

- **必须通过**：所有单项验证分数 ≥ 80 分
- **推荐通过**：综合评分 ≥ 85 分

## 常见问题

### Q1：如何确保 agent 识别为 AI 主题？

**A**：在输入信息中明确指定主题类型：
```
主题：ChatGPT 原理
主题类型：AI 技术科普
目标受众：普通读者
```

### Q2：如何选择合适的比喻？

**A**：
1. 比喻要与 AI 技术主题相关
2. 比喻要生活化、易于理解
3. 比喻要贯穿全文，形成统一的故事线
4. 常用比喻："驾驶之旅"、"副驾驶"、"指挥官"

### Q3：如何设计有效的互动话题？

**A**：
1. 互动话题要与文章内容相关
2. 提供 3-5 个明确的选项
3. 每个选项有 emoji 标记
4. 鼓励读者在评论区分享体验

### Q4：配图建议如何编写？

**A**：
1. 配图建议是描述性的，非固定链接
2. 配图建议要具体、易于想象
3. 配图建议要符合阶段主题
4. 使用 `**（配图建议：[描述]）**` 格式
